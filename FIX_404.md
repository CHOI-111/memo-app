# GitHub Pages 404 오류 해결 가이드

## 문제 진단

404 오류가 발생하는 주요 원인:
1. ❌ `index.html` 파일이 저장소에 없음
2. ❌ 파일이 하위 폴더에 있음 (루트에 있어야 함)
3. ❌ GitHub Pages 설정이 잘못됨
4. ❌ 배포가 아직 완료되지 않음

## 해결 방법

### 1단계: 저장소 확인

https://github.com/CHOI-111/memo-app 페이지에서:

1. **Code 탭** 클릭
2. `index.html` 파일이 목록에 있는지 확인
   - ✅ 있음 → 2단계로
   - ❌ 없음 → 파일 업로드 필요

### 2단계: 파일 업로드 (파일이 없는 경우)

#### 방법 A: 웹에서 직접 업로드

1. 저장소 페이지에서 **"Add file"** → **"Upload files"** 클릭
2. `index.html` 파일을 드래그 앤 드롭
3. 하단에서 **"Commit changes"** 클릭
4. 제목: "index.html 추가"
5. **"Commit changes"** 버튼 클릭

#### 방법 B: Git 명령어 사용

```bash
# 프로젝트 폴더로 이동
cd "C:\Users\CKIRUser\Documents\MyProject - 복사본"

# Git 초기화 (이미 했다면 생략)
git init

# index.html 추가
git add index.html

# 커밋
git commit -m "index.html 추가"

# 브랜치 이름 확인/변경
git branch -M main

# 원격 저장소 연결 (이미 했다면 생략)
git remote add origin https://github.com/CHOI-111/memo-app.git

# 푸시
git push -u origin main
```

### 3단계: GitHub Pages 설정 확인

1. 저장소 페이지에서 **"Settings"** 탭 클릭
2. 왼쪽 사이드바에서 **"Pages"** 클릭
3. **"Source"** 섹션 확인:
   - Branch: **`main`** 선택
   - Folder: **`/ (root)`** 선택
4. **"Save"** 버튼 클릭

### 4단계: 배포 대기

- GitHub Pages 배포는 보통 **1-5분** 정도 걸립니다
- Settings → Pages 페이지에서 배포 상태 확인 가능
- "Your site is live at..." 메시지가 보이면 배포 완료

### 5단계: 접속 확인

배포 완료 후:
- `https://choi-111.github.io/memo-app/` 접속
- 또는 `https://choi-111.github.io/memo-app/index.html` 접속

## 추가 확인사항

### 파일 이름 확인
- ✅ `index.html` (소문자)
- ❌ `Index.html` (대문자)
- ❌ `INDEX.HTML` (대문자)

GitHub Pages는 대소문자를 구분할 수 있으므로 정확히 `index.html`이어야 합니다.

### 파일 위치 확인
```
✅ 올바른 위치:
memo-app/
  └── index.html

❌ 잘못된 위치:
memo-app/
  └── public/
      └── index.html
memo-app/
  └── src/
      └── index.html
```

### 캐시 문제 해결

브라우저 캐시 문제일 수 있습니다:
1. **Ctrl + F5** (강제 새로고침)
2. 또는 **Ctrl + Shift + R**
3. 시크릿 모드에서 접속해보기

## 빠른 체크리스트

- [ ] `index.html` 파일이 저장소 루트에 있음
- [ ] 파일 이름이 정확히 `index.html` (소문자)
- [ ] GitHub Pages 설정: Branch `main`, Folder `/ (root)`
- [ ] 배포 완료까지 몇 분 대기
- [ ] 브라우저 캐시 삭제 후 재접속

## 여전히 안 되는 경우

1. **Actions 탭 확인**
   - 저장소 → **Actions** 탭
   - 배포 작업이 실패했는지 확인

2. **파일 내용 확인**
   - `index.html` 파일을 클릭해서 내용이 올바른지 확인

3. **다시 배포**
   - Settings → Pages에서 설정을 다시 저장
   - 또는 파일을 수정하고 다시 커밋

4. **URL 확인**
   - 정확한 URL: `https://choi-111.github.io/memo-app/`
   - 사용자명과 저장소명이 정확한지 확인
